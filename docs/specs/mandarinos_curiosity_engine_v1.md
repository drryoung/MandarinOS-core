
# MandarinOS Curiosity Engine v1

Purpose:
Enable natural follow‑up questions when something interesting or unfamiliar appears in conversation.

This engine works across all conversation engines:
- Identity
- Place
- Family
- Study / Work
- Food
- Travel

The Curiosity Engine has two modes:

1. Normal Curiosity
2. Emergency Curiosity (for beginners)

--------------------------------------------------

NORMAL CURIOSITY MODEL

When new information appears:

New Information
→ Clarification
→ Description
→ Reaction
→ Bridge

Example

Persona: 我是医生。
(I am a doctor.)

Possible curiosity follow‑ups:

你主要做什么？
What do you mainly do?

这个工作难吗？
Is this job difficult?

为什么？
Why?

--------------------------------------------------

EMERGENCY CURIOSITY LOOP

When the learner hears something unfamiliar, they should be able to probe the topic instead of stopping the conversation.

Goal:
Allow the learner to ask simple questions to understand the topic.

Basic loop:

Unknown Thing
→ What is it?
→ What do you mainly do?
→ Is it difficult?
→ Why?

--------------------------------------------------

CORE EMERGENCY QUESTIONS (P1)

这是什么？
What is this?

什么意思？
What does it mean?

你主要做什么？
What do you mainly do?

难吗？
Is it difficult?

为什么？
Why?

像什么？
What is it like?

--------------------------------------------------

EXPANDED CURIOSITY QUESTIONS (P2)

这个是什么意思？
What does this mean?

你可以简单说吗？
Can you explain simply?

你可以举个例子吗？
Can you give an example?

这个主要是做什么的？
What is this mainly for?

DESIGN PRINCIPLE

Curiosity is not failure.

Repair = fixing misunderstanding

Curiosity = exploring new information

MandarinOS should encourage curiosity whenever possible.
